## 🔄 数据流程

### 1. 用户发送消息

```
手机浏览器
    │
    │ POST /api/chat {"message": "你好"}
    ▼
Flask Server
    │
    │ 添加到对话历史
    ▼
构建 LLM 提示词
    │
    │ 包含最近 10 轮对话
    ▼
调用 LLM API
    │
    │ 返回 AI 回复（可能包含工具调用）
    ▼
解析工具调用
    │
    │ [TOOL:memory]search: 上周计划[/TOOL]
    ▼
执行工具
    │
    │ MemoryTool.search("上周计划")
    ▼
组合结果
    │
    │ AI 回复 + 工具结果
    ▼
返回给用户
    │
    │ {"response": "...", "timestamp": "..."}
    ▼
手机浏览器显示
```

### 2. 工具调用流程

```
LLM 生成工具调用
    │
    │ [TOOL:memory]search: 关键词[/TOOL]
    ▼
正则表达式解析
    │
    │ tool_name = "memory"
    │ params = "search: 关键词"
    ▼
路由到对应工具
    │
    ├─→ memory → MemoryTool.search()
    ├─→ db → DBTool.execute_query()
    ├─→ reflection → ReflectionTool.generate_daily()
    └─→ intelligence → IntelligenceTool.fetch_latest()
    │
    ▼
执行工具逻辑
    │
    ▼
返回工具结果
    │
    │ "记忆搜索结果：..."
    ▼
插入到 AI 回复中
```

## 📊 性能指标

### 响应时间

```
API 响应时间
├── /api/chat: 1-3 秒（取决于 LLM）
├── /api/history: < 100ms
├── /api/clear: < 50ms
└── /health: < 10ms
```

### 并发能力

```
开发环境
├── Flask 内置服务器
├── 单进程
└── 适合 1-5 个并发用户

生产环境
├── Gunicorn (4 workers)
├── 多进程
└── 适合 20-50 个并发用户
```

### 资源占用

```
内存占用
├── Flask 进程: ~50MB
├── 对话历史: ~1MB (50 条)
└── 总计: ~100MB

CPU 占用
├── 空闲: < 1%
├── 处理请求: 5-10%
└── LLM 调用: 取决于 API
```

### 2. 移动优先

## 🎉 总结

这个架构设计展示了：

1. **极简设计** - 用最少的组件实现完整功能
2. **清晰分层** - 前端、后端、工具、存储各司其职
3. **灵活扩展** - 支持多种访问方式和部署模式
4. **安全可靠** - 提供多层次的安全选项
5. **性能优化** - 合理的资源占用和响应时间
